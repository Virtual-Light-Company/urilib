Uniform Resource Identifier Library

Uniform Resource Identifiers (URI) are the parent of the more well know Uniform Resource Locators (URL). This library provides a complete implementation of both the full URI suite of functionality.

The library takes many hints from the standard java.net package and the way it presents a URL class to the user. However, in my opinion, that package misses a number of vital ingredients - the primary function being able to listen to the download progress so common is almost every net application (from FTP clients to web browsers). In these cases and where the overriding URI system comes into play, the two libraries have diverged. In general it is very close to being a pluggable replacement with only the import lines and a couple of minor API changes needed.

Dealing with URIs

URIs are much more flexible than the Java interfaces seem to imply. There is a sibling to URLs called Uniform Resource Names (URN) that gives you naming without location information. The underlying system then resolves that to a real location and gives you back the information. As a complement to this you can also change URLs to URNs and back again at will. This library provides all of these facilities.

Customisation Features

Adhering to the standard Java philosophy almost everything is replacable or overridable within the library. There are a collection of standard behaviours for which the user can provide expanded or different functionality.

Four areas can be configured:

  1. The resolver(s) to change a URL <-> URN <-> URC.
  2. The handlers for individual network protocols like HTTP and FTP
  3. The handlers for various content types like JPEG image and HTML text
  4. The mapping of file names to content types
Each of these areas can be changed in a number of ways:
  1. Providing a custom coded factory within the code.
  2. Locating the class in the package from the defined system resource
  3. Locating the class from a default package name for this library
  4. Where available, loading the class from the java.net default areas
For more information on the specifics of these properties, see the package documentation for {@link org.ietf.uri}.

What's Provided

There are two main areas in this library: The core URI mechanisms and a collection of resolvers and handlers. You will notice this in the package names.

The core URI libs are provided in the org.ietf.uri package hierarchy. These classes provide the basic capabilities of resource management, event management, URI class representations and utilities.

A collection of resolvers and handlers are provided in the vlc.net packages. These are specific implementations of Resolver Discovery Services, protocol handlers and content handlers. Not included in this deliverable, but also available from VLC are a set of content handlers specifically for loading many different image types
